# Security Policy

## Supported Version

The production deployment at `aljun-cursiga-portfolio.vercel.app` and the latest commit on `main` are supported.

## Reporting a Vulnerability

Please report suspected vulnerabilities privately by emailing:

`aljuncursiga09351522067@gmail.com`

Include:

- A clear description of the issue
- The affected URL or file
- Reproduction steps
- Potential impact
- Any suggested remediation

Please do not include secrets, personal information, or destructive proof-of-concept data. Do not open a public GitHub issue for an unpatched vulnerability.

You should receive an acknowledgement within seven days. Confirmed issues will be investigated, fixed, tested, and disclosed after a safe resolution is available.

## Scope

In scope:

- The portfolio website source code
- The public Vercel deployment
- Portfolio-owned configuration and dependencies

Out of scope:

- GitHub, Vercel, Google Fonts, or other third-party infrastructure
- Social engineering or denial-of-service testing
- Automated traffic that disrupts service
- Vulnerabilities in linked repositories unless reported to that repository's owner
